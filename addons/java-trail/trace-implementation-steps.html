Steps to Deliver JavaTrailCam

  1. Extend SimpleDebugVM so its event loop can dispatch both field watchpoints and method entry/exit events, exposing a reusable method-trace session API.
  2. Create the JavaTrailCam entry point that reads a JSON configuration via JsonHelper.parseObject and validates required parameters.
  3. Use the configuration to attach to the target JVM, register method entry/exit requests, and start tracing from the specified class and method.
  4. Track the active call stack per event thread, logging entry and exit events with class, method, signature, line number, depth, timestamps, and duration.
  5. Respect include/exclude filters and the skipOutsideDomain option while keeping the internal stack balanced for filtered calls.
  6. Stop the trace when execution returns to the starting frame or when configured limits (depth, duration, await timeout) are reached, marking the outcome accordingly.
  7. Serialise the captured trace to JSON using JsonHelper.stringifyObject, writing to the configured output file or standard output.
  8. Update project documentation to describe configuration parameters, JSON output, and the workflow for running JavaTrailCam alongside the existing JavaTrail tooling.
